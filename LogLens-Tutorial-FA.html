LogLens — راهنمای استفاده

مشاهده‌گر لاگ در مقیاس پروداکشن برای Laravel: ایندکس دائمی روی دیسک، باز کردن آنی فایل‌های چندگیگابایتی، جستجوی Full-Text، tail زنده در مرورگر، گروه‌بندی خطاها و RTL.

Laravel 8–13 PHP 8.0–8.5 ۹۴ تست سبز بدون افزونه‌ی اجباری v1.0.0

#معرفی

LogLens یک پکیج Laravel است که برخلاف ابزارهای موجود، ایندکس لاگ را به‌جای cache اپلیکیشن در یک فایل SQLite کنار هر لاگ روی دیسک نگه می‌دارد. نتیجه: باز شدن آنی فایل‌های بزرگ، جستجوی «یک‌بار ایندکس، بارها جستجو»، و مصونیت در برابر پاک شدن cache.

باز کردن آنی

جدیدترین ورودی‌ها با خواندن معکوس از انتهای فایل، بدون نیاز به ایندکس.

جستجوی FTS5

شتاب substring با توکنایزر trigram و یک زبان کوئری ساده.

Tail زنده

SSE با fallback خودکار به polling؛ بدون PCNTL، سازگار با ویندوز و Octane.

گروه‌بندی خطا

اثرانگشت قطعی برای exceptionها؛ نمای Issues مثل Sentry.

۱پیش‌نیازها

موردحداقلتوضیح
PHP8.0+تا ۸.۵ تست‌شده
Laravel8 تا 13ServiceProvider دستی، بدون migration
افزونه‌ها—pdo_sqlite، zlib، zip اختیاری با fallback
اگر pdo_sqlite نباشد، LogLens به یک ایندکس باینری فشرده fallback می‌کند و باز هم کار می‌کند (فقط جستجو کندتر می‌شود).

۲نصب

نصب واقعاً صفر-پیکربندی است؛ نه publish، نه migration، نه npm:

composer require loglens/loglens

حالا کافی است به آدرس زیر بروید (در محیط local به‌صورت پیش‌فرض باز است):

# مرورگر:
https://your-app.test/loglens

برای انتشار اختیاریِ فایل پیکربندی:

php artisan vendor:publish --tag=loglens-config
حذف کامل = حذف پکیج + پاک کردن پوشه‌ی storage/loglens/. هیچ باقی‌مانده‌ای (migration، جدول، cache key) نمی‌ماند.

۳امنیت و دسترسی

مهم: در محیط غیرِ local، LogLens به‌صورت پیش‌فرض همه‌ی درخواست‌ها را ۴۰۳ می‌کند تا وقتی gate را تعریف کنید. این یعنی production امن است مگر اینکه خودتان دسترسی بدهید.

در یک ServiceProvider (مثلاً App\Providers\AppServiceProvider) داخل متد boot:

use Illuminate\Support\Facades\Gate;

Gate::define('viewLogLens', function ($user = null) {
    return in_array(optional($user)->email, [
        'admin@example.com',
    ]);
});

کلیدهای کنترل دسترسی

Gateکاربرد
viewLogLensدسترسی کلی (وب/API/SSE یکسان)
downloadLogFileدانلود فایل
deleteLogFile / clearLogFileحذف / پاک‌سازی محتوا
pruneLogFilesاجرای prune

کلیدهای قطع سریع در config: read_only، allow_delete، allow_clear، allow_download. حذفِ secretها (توکن، APP_KEY، کلید AWS/Stripe، JWT، پسورد) به‌صورت پیش‌فرض روشن است.

۴پیکربندی

مهم‌ترین کلیدهای config/loglens.php:

return [
  'roots'   => [storage_path('logs')],     // کجا دنبال لاگ بگردد
  'route'   => ['prefix' => 'loglens', 'middleware' => ['web']],
  'api_only' => false,                    // فقط API، بدون UI
  'read_only' => false,                   // قفل همه‌ی عملیات مخرب
  'index' => [
    'directory'   => storage_path('loglens'),
    'size_budget' => '1G',                // سقف حجم ایندکس با حذف LRU
  ],
  'security' => [
    'redaction' => ['enabled' => true],  // حذف secret هنگام نمایش
  ],
  'editor' => ['default' => 'phpstorm'],   // لینک باز کردن فریم در ادیتور
  'locale' => ['default' => null],          // null = خودکار از app locale
];
فرمت‌های سفارشی را بدون نوشتن کلاس PHP می‌توانید فقط با یک regex در کلید parsing.custom اضافه کنید. جزئیات کامل در docs/configuration.md.

۵کار با رابط کاربری

چیدمان سه‌پنلی شبیه IDE:

  • سایدبار (راست): فایل‌ها/پوشه‌ها، فیلتر سطح‌ها با شمارش، و تب Issues.
  • ستون مرکزی: نوار جستجو، هیستوگرام حجم لاگ، و لیست مجازی‌سازی‌شده‌ی ورودی‌ها.
  • کشوی جزئیات (چپ): متن خام، فیلدهای پارس‌شده، درخت JSON، و stack trace با لینک ادیتور.

۶زبان جستجو

همین زبان در UI، API، و دستورات CLI کار می‌کند.

الگومثالمعنی
عبارت سادهpayment failedهر دو کلمه (AND ضمنی)
عبارت دقیق"payment failed"عین عبارت
سطحlevel:errorفقط error
مقایسه سطحlevel:>=warningwarning به بالا
زمان نسبیafter:-1hیک ساعت اخیر

یک کوئری کامل:

level:error after:-1h "payment failed" -channel:horizon
پیش‌فرض، جستجو literal است (نه regex)؛ پس [error] (1.5) دقیقاً همان متن جستجو می‌شود.

۷Tail زنده

در UI دکمه‌ی Live را بزنید تا ورودی‌های جدید لحظه‌ای بیایند. زیر کاپوت: ابتدا SSE تلاش می‌شود؛ اگر یک proxy بافر کند یا Octane باشد، خودکار به polling سوییچ می‌کند — بدون از دست رفتن یا تکرار ورودی. می‌توانید tail را با زبان جستجو فیلتر کنید (مثلاً فقط level:>=error).

۸گروه‌بندی خطاها (Issues)

به‌جای ۱۴۳۲ ردیف تکراری، یک ردیف با شمارش، اولین/آخرین رخداد و sparkline می‌بینید. اثرانگشت‌ها هنگام ایندکس و قطعی محاسبه می‌شوند، پس یک خطا روی فایل‌های روزانه‌ی مختلف هم در یک گروه جمع می‌شود. با «New since» می‌توانید خطاهای جدید بعد از یک deploy را ببینید.

۹دستورات خط فرمان

# ساخت/به‌روزرسانی ایندکس همه‌ی فایل‌ها (مثلاً در deploy hook)
php artisan loglens:index
php artisan loglens:index --watch          # حلقه‌ی پیوسته

# جستجو با همان زبان کوئری
php artisan loglens:search "level:error after:-24h timeout" --json

# tail زنده در ترمینال (روی ویندوز هم کار می‌کند)
php artisan loglens:tail --query="level:>=warning"

# خلاصه‌ی آماری هر فایل
php artisan loglens:stats

# نگه‌داری: حذف/فشرده‌سازی لاگ‌های قدیمی
php artisan loglens:prune --days=30 --max-total-size=10G --compress

برای زمان‌بندی prune در routes/console.php (یا Kernel):

Schedule::command('loglens:prune --days=30')->daily();

۱۰JSON API

تمام قابلیت‌های UI از طریق API هم در دسترس‌اند (زیرِ /loglens/api). چند نمونه:

GET  /loglens/api/files
GET  /loglens/api/files/{id}/open
GET  /loglens/api/files/{id}/search?q=payment+level:error
GET  /loglens/api/files/{id}/groups
GET  /loglens/api/files/{id}/histogram
GET  /loglens/api/tail/poll?file={id}&cursors={...}
GET  /loglens/api/diagnostics

با api_only=true می‌توانید UI را کاملاً خاموش و فقط API را فعال نگه دارید. مرجع کامل: docs/api.md.

۱۱نکات پروداکشن

  • gate را تعریف کنید وگرنه production همه‌چیز را ۴۰۳ می‌کند (همین که می‌خواهید).
  • درایور cache سالم داشته باشید (file/redis/array یا databaseِ migrate‌شده) — endpointهای rate-limit‌شده (search/tail/index) به آن نیاز دارند. مرور لاگ حتی با cache خراب کار می‌کند.
  • ایندکس پس‌زمینه: اگر queue worker زنده باشد از job استفاده می‌شود؛ وگرنه scheduler؛ وگرنه ایندکس in-request زمان‌دار. برای deploy از loglens:index استفاده کنید.
  • کانتینرها: لاگ را هم‌زمان به stderr و یک فایل بفرستید (نسخه‌ی dual-channel در docs/deployment.md).
  • Octane: بدون state اشتراکی است؛ tail خودکار به polling می‌رود.
هشدار نسخه‌ی لاراول: اگر هنگام نصب با خطای security advisory برخوردید (PKSA-…)، این سمت لاراول است نه LogLens؛ به آخرین نسخه‌ی patch‌شده‌ی لاراول بروید یا composer audit بگیرید.

نشانهعلت / راه‌حل

search/tail خطای ۵۰۰درایور cache میزبان خراب است (مثلاً database بدون migration). یک cache سالم تنظیم کنید.
فایل در حالت building می‌ماندqueue/scheduler نیست؛ یک‌بار php artisan loglens:index بزنید.
